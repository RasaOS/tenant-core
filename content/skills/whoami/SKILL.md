# /whoami — State the composed RasaOS identity

Answer "who are you?" with the full, composed identity — the one-liner that
tells anyone (human or agent) exactly what this project is the moment they ask.

The identity has **three layers** (canon SA-025); this skill reads them and
composes them into one sentence:

```
[RasaOS]      the OS for agentic AI vertical platforms   (constant)
[Element]     a <role> for <subject>                     (.claude/rasa-identity.md)
[Deployment]  <project> — <platform> for <company>       (.claude/rasa-deployment.md)
```

## Behavior contract

- **Read both identity files first.**
  - `.claude/rasa-identity.md` — the element layer, generated by `bin/init` from
    the Element's `rasa.identity` (RasaOS + `<role>` + `<subject>` + goal). If
    it's missing, this project wasn't installed from a rasa Element — say so.
  - `.claude/rasa-deployment.md` — the deployment layer, project-owned. If its
    fields are still the `[bracketed placeholders]`, the deployment layer is
    **unfilled** — surface that (the identity is partial) and offer to help fill it.
- **Compose, don't just dump.** Produce one clean sentence that reads across all
  three layers. Fall back gracefully: if the deployment layer is unfilled, give
  the element identity alone and note the missing piece.
- **Read-only.** This skill never edits. (Filling the deployment layer is the
  user editing `.claude/rasa-deployment.md`, which `/whoami` can walk them through
  on request but does not do silently.)

## Process

1. Read `.claude/rasa-identity.md` → extract `<role>`, `<subject>`, the element
   name, and goal.
2. Read `.claude/rasa-deployment.md` → extract Project / For / Platform. Detect
   unfilled `[...]` placeholders.
3. Compose and print the identity:

   **Fully filled:**
   > **I am a RasaOS `<role>` for `<subject>` — `<project>`, a `<platform>` product for `<company>`.**

   **Deployment unfilled:**
   > **I am a RasaOS `<role>` for `<subject>` (element `<name>`).** *(Deployment
   > layer not filled — edit `.claude/rasa-deployment.md` to add the project /
   > company / platform, and I'll speak the full identity.)*

4. Optionally follow with the layered breakdown (RasaOS / Element / Deployment)
   for detail, and the pinned version from `rasa-identity.md`.

## What NOT to do

- **Don't invent the deployment layer.** If `.claude/rasa-deployment.md` is
  unfilled, say the identity is partial — never fabricate a project/company/platform.
- **Don't edit either identity file silently.** `/whoami` is read-only; filling
  the deployment layer is the user's edit (walk them through it only on request).
- **Don't answer from memory or the repo name.** The identity is what the two
  files say — read them; if `.claude/rasa-identity.md` is missing, say so rather
  than guessing.

## Done when

The user has the single composed identity sentence, plus (if asked) the layered
breakdown — and, if the deployment layer was unfilled, a clear pointer to fill it.
